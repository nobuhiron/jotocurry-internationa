# Brandページ コーディングプラン

## 概要
TOPページとは異なるBrandページを作成します。スマホ優先で、PCサイズとレスポンシブ対応を行います。

## ページ構成

### 1. ページファイル
- **ファイル**: `src/pages/brand.astro`
- **役割**: Brandページのセクションを並べる
- **使用Layout**: `Layout.astro`（共通）

### 2. セクション構成（7セクション）

#### セクション1: ヒーローセクション（「上等カレーとは」）
- **ファイル**: `src/sections/SectionBrandHero.astro`
- **スタイル**: `src/styles/sections/p-brand-hero.css`
- **内容**:
  - 大きなヒーロー画像（店舗外観のプレースホルダー）
  - タイトル「上等カレーとは」
  - 説明文（複数段落）
- **レイアウト**: スマホは縦並び、PCは横並び（画像とテキスト）

#### セクション2: 起源と味へのこだわり
- **ファイル**: `src/sections/SectionBrandOrigin.astro`
- **スタイル**: `src/styles/sections/p-brand-origin.css`
- **内容**:
  - セクションタイトル「上等カレーの起源と味へのこだわり」
  - 2カラムレイアウト
    - 左：カレーうどんの画像（プレースホルダー）+ 説明文
    - 右：シェフの画像（プレースホルダー）+ 説明文
- **レイアウト**: スマホは縦並び、PCは2カラム

#### セクション3: 唯一無二の味づくり
- **ファイル**: `src/sections/SectionBrandUnique.astro`
- **スタイル**: `src/styles/sections/p-brand-unique.css`
- **内容**:
  - セクションタイトル「唯一無二の味づくり」
  - 2カラムレイアウト
    - 左：複数のテキストブロック（3つの段落、各段落の前にオレンジの縦線）
    - 右：大きな画像（多数のポット、プレースホルダー）
- **レイアウト**: スマホは縦並び、PCは2カラム（画像右）

#### セクション4: 豊富なメニュー
- **ファイル**: `src/sections/SectionBrandMenu.astro`
- **スタイル**: `src/styles/sections/p-brand-menu.css`
- **内容**:
  - セクションタイトル「幅広いニーズに応える豊富なメニュー」
  - 説明文
  - 大きなメニュー画像（カツカレー、プレースホルダー）
- **レイアウト**: 中央揃え

#### セクション5: 人気メニュー紹介
- **ファイル**: `src/sections/SectionBrandPopularMenu.astro`
- **スタイル**: `src/styles/sections/p-brand-popular-menu.css`
- **内容**:
  - セクションタイトル「人気メニュー紹介」
  - グリッドレイアウト（6つのメニューカード）
    - 各カード：画像（プレースホルダー）+ メニュー名 + 価格
- **レイアウト**: スマホは1列、タブレットは2列、PCは3列

#### セクション6: 歴史への歩み
- **ファイル**: `src/sections/SectionBrandHistory.astro`
- **スタイル**: `src/styles/sections/p-brand-history.css`
- **内容**:
  - セクションタイトル「上等カレーの歴史への歩み」
  - タイムライン形式
    - 1970年代：画像 + 説明文
    - 1980年代：画像 + 説明文
    - 現在：画像 + 説明文
- **レイアウト**: 縦並びタイムライン（スマホ・PC共通）

#### セクション7: 世界の店舗
- **ファイル**: `src/sections/SectionBrandStores.astro`
- **スタイル**: `src/styles/sections/p-brand-stores.css`
- **内容**:
  - セクションタイトル「世界の店舗」
  - カルーセル（Splide使用）
    - 3つの店舗画像（プレースホルダー）
    - 各画像下にキャプション
- **レイアウト**: カルーセル（スマホ・PC共通）
- **注意**: Splideライブラリのインストールが必要

## スタイリング方針

### CSS構成
- 各セクションのスタイルは `src/styles/sections/` 配下に配置
- 命名規則: `p-brand-[section-name].css`
- FLOCSS + BEM 構成を厳守

### レスポンシブ対応
- スマホ優先（モバイルファースト）
- ブレークポイント: `--md` (768px), `--lg` (1024px)
- `clamp()` を使用した fluid typography
- Grid / Flex を使用した自動調整レイアウト

### 画像
- すべてプレースホルダーで開始
- プレースホルダーは `background-color` + `background-image` のグラデーションで表現
- アクセシビリティ: 意味のある画像には `alt` 属性、装飾画像は `aria-hidden="true"`
